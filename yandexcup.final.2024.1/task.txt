Соединение частей артефактов — сложная и кропотливая задача.
Для решения этой задачи бюро обратилось к вам — нужно разработать систему учёта артефактов.

Ваша система должна позволять:

Регистрировать новые части, найденные во время раскопок, и создавать для них новые артефакты,
если они ещё не ассоциированы с каким-либо существующим артефактом.
Объединять артефакты, когда выявлено, что их части являются фрагментами одного целого.
Например, если элементы барельефа объединяются в единую картину.
При объединении всех частей собирать их в новый артефакт,
при этом предыдущие два артефакта должны считаться устаревшими и более не использоваться.
Запрашивать информацию о составе артефакта, предоставляя список всех частей,
из которых он состоит, для дальнейшего анализа и изучения.
Помогите археологам собрать воедино разбросанные по пескам времени элементы истории и с помощью вашей системы восстановить великие объекты древних культур.

Формат ввода
Это интерактивная задача. При запуске решения на стандартный поток ввода поступают запросы одного из четырёх видов:

CREATE <part_id> — завести в системе новый артефакт, состоящий из части с идентификатором part_id;
MERGE <artefact_id1> <artefact_id2> — объединить все части, из которых состоит артефакт с идентификаторами artefact_id1 и artefact_id2. После этого артефакты перестают существовать в системе;
GET_PARTS <artefact_id> — вывести все части, из которых состоит артефакт с идентификатором artefact_id;
EXIT — завершить работу.
part_id — строка из не более чем 20 строчных английских букв и цифр от 0 до 9. Общее количество запросов без учета EXIT не превосходит 10\^5.
Количество запросов GET_PARTS не превосходит 100.

Формат вывода
На запрос CREATE <part_id> — artefact_id нового артефакта,
если артефакт, включающий в себя часть part_id, ещё не был создан, «ERROR: EXISTING PART» в противном случае.
Формат artefact_id — строка из не более, чем 20 строчных латинских букв и цифр;
На запрос MERGE <artefact_id1> <artefact_id2> — artefact_id объединённого артефакта.
artefact_id1 и artefact_id2 далее в запросах использоваться не будут;
На запрос GET_PARTS <artefact_id> — вывести все part_id через пробел.
Если артефакта с таким идентификатором не существует, то вывести «ERROR: NO SUCH ARTEFACT».
Гарантируется, что суммарно будет выведено не больше, чем 2*10\^5 part_id.

Пример взаимодействия

Server:
CREATE p1
CREATE p2
MERGE paws tail
CREATE p3
MERGE whiskers body
GET_PARTS cat001
EXIT

Client:
paws	
tail
body
whiskers
cat001
p1 p2 p3
